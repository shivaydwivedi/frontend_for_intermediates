Modal dialog

Use this modal to edit fictional display preferences. Focus is moved inside, trapped while open, and restored to the opener after closing.

Keyboard and accessibility

  • Tab and Shift+Tab remain inside the dialog.
  • Escape and explicit close controls close it.
  • The opener receives focus after close when it still exists.
const modal = createModal(dialog);
modal.open(button);

Tabs

Use roles only for the custom tab relationship. The buttons remain native buttons.

Only the active tab is in the normal Tab sequence. Selection updates the panel.

Tests cover roving tabindex, wrapping, activation, and panel visibility.

Keyboard and accessibility

  • Left and Right arrows move focus.
  • Home and End move to the first and last tabs.
  • Enter or Space activates the focused tab.
const tabs = createTabs(root);
tabs.select(1);

Accordion

Headings contain buttons, so Enter and Space work without custom keyboard code.

The button's expanded state and panel visibility update together.

Panels are simple content containers here, so no extra region roles are needed.

Keyboard and accessibility

  • Enter and Space toggle through native button behavior.
  • Opening a panel does not move focus.
  • Multiple panels may be open.
const accordion = createAccordion(root);
accordion.toggle(0);

Menu button

Keyboard and accessibility

  • This is an action menu, not navigation.
  • Arrow keys move between actions.
  • Escape closes and restores focus.
const menu = createMenuButton(root);
menu.open('first');

Toast notifications

Keyboard and accessibility

  • Toasts do not steal focus.
  • Messages are rendered as text.
  • Error toasts use assertive announcement in this demo.
toastRegion.showToast('Saved', {
  tone: 'success'
});

Tooltip

Only one item in a composite widget uses tabindex="0"; arrow keys move focus among the rest.

Keyboard and accessibility

  • Focus and hover reveal the tooltip.
  • Escape hides it.
  • Tooltip content is supplemental, not required.
const tooltip = createTooltip(root);
tooltip.show();

Editable combobox

    Custom values are allowed. Suggestions use safe text rendering.

    Keyboard and accessibility

    • Typing filters suggestions case-insensitively.
    • Arrow keys move the active descendant while focus stays in the input.
    • Enter accepts the active suggestion; Escape closes the list.
    const combobox = createCombobox(root, {
  suggestions
});

    Disclosure navigation

    The page header uses the disclosure navigation component. Links remain ordinary links, and JavaScript only enhances the narrow-screen collapse behavior.

    Keyboard and accessibility

    • The toggle is a native button.
    • Navigation is a semantic nav, not a menu widget.
    • Escape closes the narrow-screen disclosure.
    const nav = createDisclosureNavigation(root);
nav.sync();
